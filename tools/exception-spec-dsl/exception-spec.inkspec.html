Per-deployment exception spec

The single editable place a Forward Deployed Engineer authors the business logic that becomes agent behavior for one customer: decision rules (condition → action → terminal state), tolerances, exception paths and exceptions-of-exceptions, hand-to-human fallbacks, and the approval-gate config. It renders the spec readably and exports a valid fixtures/retail_ops/policy.json the eval pack and MCP kit consume unchanged — author here, export, run under it.

Round-trip contract — the loader (rules.load_policy) reads ten keys only; every scalar is a {token, value, unit} object and only .value is consumed. Everything richer the FDE authors here (the rule table, exception guards, fallbacks, per-customer overrides) rides along in a _exception_spec metadata block the loader keeps in .raw and ignores — nothing is lost, the file still loads. gated_actions is drift-guarded: it must equal the eleven-item rules.GATED_ACTIONS frozenset. Autosaves locally.
