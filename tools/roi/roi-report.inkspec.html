Automation ROI — the FDE's numbers

A business-impact readout for a supply-chain exception-automation agent, computed from a synthetic before/after ops dataset. The whole point is the honest-net discipline: a false-exception is not free — an agent that flags or auto-closes a clean transaction creates a human touch that would not otherwise exist, so false positives dock touches-saved. net_touches_saved = correct terminal auto-work − (rework from false positives + nuisance flags + escalations that still needed a human). Never a vanity number.

Target vs truth. The “40% efficiency / 4-week ROI” vendor framing is a target this report can display — never baked-in ground truth. Every headline is computed from ops_baseline.jsonl + ops_automated.jsonl + cost.json by roi.py; the eval scorecard (eval/retail_ops/history/*.json) supplies the accuracy & safety-gate cross-reference. Fully synthetic (MARKER_* tokens); see PROVENANCE.md.
